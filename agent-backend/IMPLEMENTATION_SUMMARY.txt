BEHAVIORAL CHANGES
================================================================================

BEFORE:
- Loop only stopped when LLM returned ActionType.DONE
- Naive repetition detection (only 3 consecutive identical action types)
- No completion detection (relied on LLM or max steps)
- Could repeat indefinitely if LLM kept deciding non-DONE actions

AFTER:
- Loop stops on either LLM DONE or generic completion/repetition detection
- Smart action signature tracking (type, selector, value, key)
- Page state monitoring (URL + title hash) for no-progress detection
- Generic goal-based completion heuristics:
  * Search goals: URL change + result keywords + result elements
  * Navigation goals: target domain in URL + page content
  * Generic keywords: goal words appearing in page/URL
- No more website-specific logic (works across any site)
- Returns stop_reason in result dict for debugging

EXAMPLE BEHAVIORS:
1. Search task: "search for laptop on daraz"
   - User navigates to daraz.pk
   - Searches for "laptop"
   - Page shows results with product links
   → Agent stops automatically (no DONE action needed)

2. Repetition detected: "click this button"
   - Agent clicks button 3 times
   - Page state doesn't change
   → Agent stops with "Action click repeated 3 times"

3. No progress: Typing in search box
   - Agent types search query
   - Hits Enter
   - Types again on same URL (no new page)
   → Agent stops with "No page state change detected"
