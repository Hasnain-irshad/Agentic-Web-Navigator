# ==============================================================================
# DATA FLOW EXAMPLE
# ==============================================================================

User Goal: "Find the latest Python news on GitHub"

Step 1: OBSERVE
  ├─ Extract page: https://example.com
  ├─ Find elements: buttons, links, inputs
  ├─ Generate description:
  │  "Search page with search box and language filters"
  └─ Return Observation object

Step 2: REASON
  ├─ Send to LLM:
  │  "Goal: Find Python news on GitHub"
  │  "Current: On example.com home page"
  │  "Elements: [1] Search box, [2] GitHub logo link, ..."
  ├─ LLM decides: CLICK "GitHub logo link"
  └─ Return Action object

Step 3: EXECUTE
  ├─ Find element with selector "GitHub"
  ├─ Click it
  ├─ Wait for page load
  └─ Return ActionResult

Step 4: STORE
  ├─ Store in memory:
  │  "Step 1: CLICK GitHub → Success"
  ├─ Update history
  └─ Continue loop

Step 5: REPEAT (Steps 1-4)
  ...multiple iterations...

Final: DONE
  ├─ LLM says: "Found Python trending projects"
  ├─ Break loop
  └─ Return summary with all steps

Browser Configuration
├─ HEADLESS              [Default: true] Show/hide browser
├─ BROWSER_TIMEOUT       [Default: 30000ms] Action timeout
└─ OBSERVATION_MAX_ELEMENTS [Default: 50] Max elements per page

Agent Configuration
└─ MAX_STEPS             [Default: 20] Maximum steps per task

# ACTIONS SUPPORTED
# ==============================================================================

NAVIGATION
├─ GOTO     - Navigate to URL
│  Input: {"action_type": "goto", "value": "https://example.com"}
│  Output: ActionResult with new URL
│
└─ BACK     - Go back in history
   Input: {"action_type": "back"}
   Output: ActionResult with previous URL

INTERACTION
├─ CLICK    - Click element
│  Input: {"action_type": "click", "selector": "Search button"}
│  Output: ActionResult with new page state
│
├─ TYPE     - Type text into input
│  Input: {"action_type": "type", "selector": "search", "value": "query"}
│  Output: ActionResult
│
└─ SCROLL   - Scroll page
   Input: {"action_type": "scroll", "direction": "down"}
   Output: ActionResult

TAB MANAGEMENT
├─ NEW_TAB  - Open new tab
│  Input: {"action_type": "new_tab"}
│  Output: ActionResult with tab count
│
└─ CLOSE_TAB - Close current tab
   Input: {"action_type": "close_tab"}
   Output: ActionResult with remaining tabs

CONTROL
└─ DONE     - Complete task
   Input: {"action_type": "done"}
   Output: Success, loop terminates

# ==============================================================================
# ERROR HANDLING STRATEGY
# ==============================================================================

Level 1: Action Execution Errors
  Action fails (element not found, timeout, etc)
  → Log error
  → Return ActionResult(success=False)
  → LLM sees failure, adjusts next action
  → Loop continues

Level 2: LLM Reasoning Errors
  JSON parse error, invalid action
  → Log error
  → Retry with error context (max 3x)
  → If still fails: return DONE action
  → Gracefully terminate

Level 3: Browser Errors
  Playwright crashes, connection fails
  → Log error
  → Attempt cleanup
  → Break loop
  → Return error status

Level 4: Configuration Errors
  Missing API key, invalid settings
  → Validate on startup
  → Provide clear error message
  → Exit cleanly

All errors are:
  ✓ Logged with timestamps
  ✓ Reported in task summary
  ✓ Non-blocking (loop continues)

Scalability
